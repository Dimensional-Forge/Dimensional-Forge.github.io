Bridge Roster
Module Inventory
Bridge Roster
18 modules  ·  master + 17 child
Master Control Hub · Orchestration · Control Plane
Master Control
Central bridge registry and orchestration hub. Auto-discovers each child bridge from its @bridge header, probes it via HTTP GET /tools on startup to build the unified tool registry, and proxies every call. Streams serial events and injected messages to venn.html via SSE. Exposes whitelisted run_command + file tools natively.
Auto-Discovery HTTP /tools Probe SSE Event Stream
The Agentic Core Learning & Adaptation
Memory
3-tier persistent memory: USER.md profile (100-token hard cap, always injected), long-term MEMORY.md (keyword search, up to 3 recall results), and daily session logs. Total overhead ceiling: 280 tokens.
100 Tok Profile Cap 280 Tok Ceiling Keyword Recall
Knowledge
On-device RAG pipeline. Ingests documents on startup, paragraph-chunks them into chunks.json, and serves hybrid BM25 + cosine search with RRF merge. Gracefully degrades to BM25-only if the embedding server is offline.
Hybrid BM25 + RAG RRF Merge 4 Tools
Obsidian
11-tool vault interface with zero external dependencies. Auto-detects vault path across platforms (Android + Windows). Full frontmatter support, wikilink-aware, backlink discovery, tag search, and daily note management.
11 Vault Tools Zero Dependencies Wikilinks + Backlinks
Reflection
Identity management bridge. VENN can propose updates to its own VENN.md identity document (preferences, self-description, learned behaviors) but all changes require explicit human approval before being written to disk.
Identity Updates Approval Required VENN.md Writer
Heart
Proactive pulse engine. Fires scheduled check-ins (morning anchor, hyperfocus breaker, evening reflection) into the chat pipeline via the master /ingest endpoint — VENN speaks up on its own, not just on request. Focus-timer and pause controls; all writes propose/execute token-gated.
Proactive Pulse Routine Scheduler Propose → Execute
Business Operations Execution & Quoting
Google
OAuth2 integration for Google Calendar and Gmail. Loopback authentication server on 127.0.0.1:8085 for desktop auth flow. Token persisted locally. Supports event CRUD, email read/send, and calendar listing — write tools propose/execute gated.
OAuth2 Loopback :8085 Calendar + Gmail
Timeline
Generates polished HTML project timeline estimates — Gantt-style phases, deliverables, hours, and risk assessments. Outputs standalone HTML files to brain/outputs/timeline/ that can be sent directly to clients.
7 Tools HTML Gantt Output Client-Ready Export
Print Estimator
Estimates 3D print cost and time from an STL file. Parses geometry, calculates volume and weight, looks up material pricing, applies machine and labor rates, and exports a formatted HTML quote to brain/outputs/print-estimator/.
6 Tools STL Parser HTML Quote Export
Presence & Observability Security & Comms
Voice SAM
Text-to-speech using SAM (Software Automatic Mouth, 1982). Produces characteristic retro robotic synthesis. Returns WAV audio as base64, playable via HTMLAudioElement with instant interrupt support.
SAM 1982 Engine WAV Base64 200–500ms Latency
Voice Piper
High-quality offline TTS using Piper with locally cached voice models. Multiple accent/voice options. Slower than SAM but natural speech quality. Fully offline once model is downloaded.
Piper TTS Local Voice Models Offline Capable
Security
On-device face-recognition monitoring. Enrolls authorized faces locally, watches the camera stream, and on an unknown person triggers a red-alert flash, red eyes/mouth, a spoken warning, and a timestamped transcript log. No cloud, no image upload — frames never leave the device.
Face Recognition On-Device Red-Alert + Voice
Telegram
Send and receive Telegram messages through VENN. File attachments are auto-routed into input buckets by type: STL files go to the print-estimator queue, documents, photos, voice, and video each get their own folder.
3 Tools Input Bucket Routing Send + Receive
Projection
Opens the OS screen-cast panel — Miracast on Windows, Cast settings on Android via Termux. Propose/execute gated with a short-lived confirmation token so a small local model can't fire it by accident.
Miracast / Cast Propose → Execute Win + Android
Edge Hardware Field I/O
IDE Bridge
Filesystem and shell bridge for the dev environment. Exposes read_file, write_file, list_dir, and run_command with a strict allowlist. Isolated from Master Control so filesystem access can be toggled independently.
4 Tools Whitelist Enforced R/W/List/Run
ESP32
Hardware bridge for Arduino/ESP32 devices over UART serial at 115200 baud. Accepts natural language LED commands, exposes GPIO control, and handles Arduino sketch compile and upload.
UART 115200 8N1 NL LED Control Compile + Upload
Android
Full Android device control via Termux on a rooted phone. App install/launch/kill, screen control, WiFi/BT/Airplane toggle, file system access, simulated input (tap, key, text), notifications, and Shizuku permission grants.
Termux Native Shizuku Perms Full Device Control
Flipper Zero
22 tools covering all Flipper subsystems over USB-CDC serial at 115200 8N1 with Unleashed firmware. NFC detect/dump/emulate, Sub-GHz RX/TX, IR learn/transmit, BadUSB payload upload, GPIO R/W, and storage access.
22 Tools USB-CDC 115200 8N1 NFC / Sub-GHz / IR
Browser-Side In-Tab · Not Node Bridges
Browser · In-Tab
Wake-Word
Offline "Hey Venn" wake-word detection running as a vosk WASM Web Worker inside the browser tab — not a Node bridge. Audio never leaves the device, costs zero tokens, and falls back to push-to-talk. Arms/disarms independently of the TTS pipeline.
Vosk WASM 0 Tokens Fully Offline
Browser · Full-Page
Skills
Full-page browser experiences the LLM can launch through the tool loop — e.g. Motorcycle Mode's live telemetry dashboard (GPS speed, lean angle, G-force). Pure HTML/JS using device sensors; no server-side component.
Full-Page Modes Device Sensors No Server

Data Platform Patterns
9 VENN components  ·  9 enterprise platform equivalents  ·  ADF · APIM · Medallion · Hot/Warm/Cold · Serverless · App Insights · ADF Triggers · Event Hubs · Idempotent Writes
VENN Component
Master Control
Central registry. Discovers all bridge tools on startup via SSE ingest. Routes every tool call through a single control plane — no bridge is ever called directly.
≈
Enterprise Pattern
Pipeline Orchestrator
One control plane manages pipeline activities across linked execution targets. Same pattern: discover what's available, sequence the work, enforce the contract.
Azure Data Factory Same discovery → sequence → execute pattern. One control plane, many isolated execution targets.
VENN Component
Bridge Protocol
Every bridge exposes the same two-endpoint contract: GET /tools returns a normalized schema; POST /execute runs the tool. The LLM never calls a bridge directly.
≈
Enterprise Pattern
Microservice Interface Contract
Normalized interface, isolated execution, auto-discovered schema. Each service exposes a standard contract — callers never depend on implementation details.
Azure Functions + APIM The GET /tools + POST /execute contract is the same normalized interface + isolated execution that APIM enforces.
VENN Component
RAG Pipeline
Raw files ingested on startup → paragraph-chunked and stored as chunks.json → hybrid BM25 + cosine retrieval (RRF merge). Three clean, discrete transformation stages.
≈
Enterprise Pattern
Bronze → Silver → Gold
Medallion architecture: raw landing zone, cleaned and structured layer, business-ready analytical output. Same three-stage discipline — each layer has one job.
ADLS Gen2 → ADF → Synapse Three-stage transformation with schema isolation between layers — raw files are never queried directly.
VENN Component
Memory System
USER.md — 100-token hard cap, always hot. MEMORY.md — keyword search, warm on demand. Sessions/ — daily logs, cold archive, never injected into context.
≈
Enterprise Pattern
Hot / Warm / Cold Storage
ODS for hot operational access, data lake for warm analytical queries, archive tier for compliance and audit logs. Latency vs cost drives tier placement.
ODS · Data Lake · Archive Tier placement driven by access frequency, latency budget, and per-call token cost — same tradeoffs as cloud storage tier selection.
VENN Component
Token Economics
99.2% overhead reduction (23,492 → 191 tok). LLM calls list_bridges first, starts only what the current task needs, then stops it. Nothing runs idle.
≈
Enterprise Pattern
Serverless to Zero
Azure Functions consumption plan scales to zero when idle. You pay only for what executes, not for what's registered. Same resource governance discipline.
Azure Functions Consumption Plan Dynamic discovery cuts standing tool overhead from 400–2,000 tok/call to 35–85 tok — same discipline as consumption-plan billing.
VENN Component
Debug Panel
Every module logs to a single panel with [module] prefix. Timestamps, error levels, live token stats. Enforced as a written code standard — DEBUG_GUIDE.md is the runbook.
≈
Enterprise Pattern
Structured Logging + Alerting
Centralized, tagged, queryable logs with threshold-based alerting. Every service emits to the same pipeline. Observability is an enforced engineering standard, not an afterthought.
Azure Monitor · App Insights DEBUG_GUIDE.md enforces log contracts per module — observability as an engineering standard, not a post-hoc add-on.
VENN Component
Heart Bridge
Cron-like scheduler. Declarative {schedule, prompt} routines (daily HH:MM or interval) fire into the master /ingest endpoint — the scheduler triggers the pipeline, it doesn't do the work. Every fire appends to pulse-log.jsonl.
≈
Enterprise Pattern
Pipeline Triggers
Time- and event-based triggers kick off pipeline runs on a schedule, decoupled from the activities they launch. Scheduling is its own concern, separate from execution.
Azure Data Factory Triggers · Event Grid Scheduling separated from execution — declarative jobs firing a central ingest, with append-only run history. Same shape as a trigger-driven orchestrator.
VENN Component
Telegram Bridge
External API source. Inbound messages and files are classified by type and routed into typed staging folders under brain/inputs/ — landing zones downstream workers (e.g. print-estimator) pick up from. Credentials are env-first and gitignored.
≈
Enterprise Pattern
Streaming Ingest + Landing Zone
Real-time events land in raw, type-partitioned staging before any transform. Classify-on-arrival, never process in place; downstream consumers read from the zone.
Event Hubs · ADLS Landing Zone Real-time API ingestion with classify-on-arrival routing into staging — the integration edge of a hub, plus credentials-as-config.
VENN Component
Tool Gate (Propose/Execute)
Write-side tools split into a propose_* step that validates and returns a one-shot token, and a run_* step that only executes against a valid, unconsumed token. Schema validation up front, idempotency in the middle, fuzzy-key recovery as a safety net.
≈
Enterprise Pattern
Data Quality + Idempotent Writes
Validate-before-write gates plus exactly-once semantics so retries don't double-apply. Side effects are guarded by a single-use claim, not best-effort.
Data Quality Gates · Idempotent Pipeline Writes Validation gate + one-shot token = idempotent, exactly-once-style side effects — directly relevant to safe pipeline writes.
